NaverBlogAuto 분리 버전 안내서
=============================

목적
----
기존 통합 앱을 기능별 프로그램처럼 배포하기 위한 작업 폴더입니다.
원본 루트 파일은 그대로 두고, split_version 폴더 안에서만 분리 실행 구조를 관리합니다.


구성
----
app.py
  공통 앱 코드입니다.
  APP_MODE 값에 따라 find / engage_write 화면만 노출합니다.

app_find.py
  "고객을 찾아볼게요" 전용 실행 파일입니다.
  키워드 검색과 특정 블로거 링크 기반 서이추 기능을 포함합니다.

app_engage_write.py
  "고객과 친해질게요 + 고객을 설득할게요" 전용 실행 파일입니다.
  공감, 댓글, AI 댓글 변형, 글쓰기, 즉시 발행, 예약 발행 기능을 포함합니다.

build_split.py
  기능별 앱을 따로 빌드하는 스크립트입니다.

build.py
  호환용 전체 빌드 진입점입니다.
  실행하면 find / engage_write 2개를 모두 빌드합니다.


개발 실행
---------
아래 명령은 프로젝트 루트가 아니라 split_version 폴더에서 실행합니다.

python app_find.py
python app_engage_write.py


빌드
----
개별 빌드:

python build_split.py find
python build_split.py engage_write

전체 빌드:

python build_split.py all

또는:

python build.py


출력물
------
빌드 결과는 split_version/dist 안에 생성됩니다.

예상 이름:

NaverBlogAuto-Find
NaverBlogAuto-EngageWrite

macOS 환경에서는 .app 번들로 생성될 수 있습니다.


주의
----
이 폴더의 app.py는 원본 app.py의 복사본을 기반으로 합니다.
공통 엔진 모듈(auth, browser, engagement, scraper, posting, content 등)은 원본 프로젝트 루트의 모듈을 import해서 사용합니다.

따라서 기능 엔진을 수정해야 할 때는 원본 공통 모듈을 먼저 고치고, 필요하면 split_version/app.py의 UI 연결부만 동기화합니다.

설정, 세션, API 키, 네이버 계정 정보는 기존 앱과 같은 사용자 데이터 위치를 사용합니다.
즉, 분리 앱 2개가 같은 로그인/설정 정보를 공유합니다.


검증 명령
---------
python -m py_compile app.py app_find.py app_engage_write.py build.py build_split.py
python build_split.py --help


기능별 포함 범위
---------------
찾아볼게요:
- 키워드 기반 블로거 검색
- 특정 블로거 링크/ID 기반 공개 이웃 목록 수집
- 서로이웃 신청

친해질게요 + 설득할게요:
- 이웃 새글 기반 공감
- AI 댓글 작성
- 공감 + 댓글 통합 실행
- 키워드 기반 AI 글 생성
- 마크다운 파일 발행
- 즉시 발행
- 예약 발행
- 임시 저장
