# 송이 자료조사원 MVP Phase 재정의

작성일: 2026-05-18

## 결정 요약

- 송이는 윤미에게 전달하는 보조 기능이 아니라, 독립 실행 가능한 자료조사 직원으로 둔다.
- 윤미는 이후 다른 직원으로 교체될 예정이므로, 송이의 산출물은 특정 직원명에 묶지 않은 범용 리서치 브리프로 만든다.
- 내일 라이브 시연 목표는 외부 유료 AI 호출 없이, 실제 URL을 읽고 즉시 분석 결과와 복사용 브리프가 나오는 상태다.
- AI Council 검토 결과, 데모 안정성을 위해 1차 MVP는 결정론적 분석, JSON 저장, YouTube oEmbed/일반 웹 HTML 메타데이터 읽기, 복사 가능한 브리프를 우선한다.

## Phase 0. 데모 안정화 기준 확정

목적:
송이가 내일 바로 눌러볼 수 있는 최소 제품인지 판단하는 기준을 고정한다.

범위:
- 유료 API 호출 없음
- 외부 크롤링 의존 없음
- 로그인 사용자가 웹 콘솔에서 직접 실행
- 입력값은 URL, 제목, 원문/메모, 카테고리, 태그
- URL이 있으면 서버가 공개 웹 링크를 직접 읽어 제목, 설명, 본문 일부를 추출한다.
- YouTube 링크는 API 키 없이 oEmbed를 우선 사용해 제목과 채널 정보를 빠르게 읽고, 공개 자막 트랙이 텍스트로 내려오면 스크립트까지 읽는다.
- YouTube 자막이 서버에서 빈 응답으로 내려오거나 공개 자막이 없으면 제목/채널 분석으로 fallback한다.
- 작업 탭의 송이 1차 입력 UI는 PRD 기준에 맞춰 URL 입력 중심으로 둔다.

산출물:
- 송이 직원 상태를 `available`로 표시
- 실행 방식은 `web_module`
- 필수 설정값 없음
- 윤미 의존성 제거
- 작업 탭에서 예리/현주와 같은 직원 선택 흐름으로 실행

검증 기준:
- 직원 목록에서 송이가 준비중이 아니라 실행 가능한 직원으로 보인다.
- 버튼 문구가 작업 탭의 송이 작업 폼으로 이어진다.
- 윤미 이름 없이 리서치 브리프가 생성된다.

## Phase 1. 송이 자료 수집 콘솔

목적:
운영자가 자료를 프로젝트 단위로 모으고, 라이브 중에도 빠르게 추가할 수 있게 한다.

범위:
- 작업 탭 안에 송이 자료조사 작업 폼 추가
- 프로젝트 생성 및 선택
- URL 입력
- 다중 URL 줄바꿈 입력
- YouTube, Instagram, TikTok, Blog 등 플랫폼 추정
- YouTube 및 일반 웹페이지 실제 읽기
- transcript 분석은 내부/보조 경로로만 유지하고, 1차 작업 UI에는 노출하지 않는다.

산출물:
- `/app` 안의 `작업` 탭 송이 작업 폼
- 프로젝트 목록
- 자료 카드 목록
- 선택 자료 상세 보기

검증 기준:
- 새 프로젝트를 만들 수 있다.
- 같은 프로젝트에 여러 자료를 추가할 수 있다.
- 추가한 자료가 새로고침 뒤에도 유지된다.
- 공개 YouTube 링크만 넣어도 제목과 채널 정보가 들어간 자료 카드가 생성된다.
- 공개 YouTube 자막을 읽을 수 있는 경우에는 스크립트가 원문에 포함된다.
- 일반 웹페이지 링크만 넣어도 title, description, 본문 일부가 들어간 자료 카드가 생성된다.
- Reels/Shorts는 실제 SNS 접근이 막힐 수 있으므로 업로드 transcript나 자막 텍스트를 원문/메모에 넣는 방식을 데모 안전 경로로 둔다.

## Phase 2. 송이 자동 정리 엔진

목적:
입력된 자료를 즉시 리서치 카드로 바꾸어 후속 콘텐츠 제작자가 사용할 수 있게 한다.

범위:
- 링크에서 읽은 제목/설명/본문 일부 추출
- 공개 YouTube 자막/업로드 transcript 기반 스크립트 추출
- 요약 추출
- 키워드/태그 정리
- 후킹 포인트 추정
- 카피 단서 추정
- 구조 단서 추정
- 성과/반응 단서 추정

산출물:
- 자료별 `summary`
- `tags`
- `insights`
- `hook_note`
- `copy_note`
- `structure_note`
- `performance_note`

검증 기준:
- URL만 넣어도 플랫폼 태그와 링크 읽기 상태가 붙는다.
- 작업 UI는 URL만 입력해 분석하는 흐름을 우선한다.
- 메모나 원문을 넣으면 반복 단서와 핵심 문장이 추출된다.
- 사용자가 메모와 태그를 수정할 수 있다.

## Phase 3. 범용 리서치 브리프 생성

목적:
송이의 결과물을 특정 직원이 아니라 다음 작업자 누구에게나 넘길 수 있는 형태로 만든다.

범위:
- 프로젝트 단위 브리프 생성
- 핵심 자료, 공통 태그, 자료별 요약 정리
- 클립보드 복사 버튼 제공

산출물:
- Markdown 형식의 리서치 브리프
- `브리프 복사` 버튼

검증 기준:
- 브리프에 프로젝트명, 자료 개수, 공통 태그가 들어간다.
- 자료별 URL, 요약, 인사이트가 포함된다.
- 윤미 이름이 포함되지 않는다.

## Phase 4. 라이브 시연 시나리오

목적:
내일 시연에서 설명과 조작 순서가 끊기지 않게 한다.

진행 순서:
1. AIMAX 콘솔 로그인
2. 직원 목록에서 송이가 `사용 가능` 상태인 것 확인
3. 송이 카드의 `작업 지시하기` 실행
4. `작업` 탭에서 송이 선택 후 새 프로젝트 생성
5. YouTube 또는 일반 웹페이지 URL 입력
6. `송이에게 지시` 클릭
7. 요약, 태그, 인사이트 자동 생성 확인
8. 태그나 메모를 직접 수정
9. `브리프 복사`로 후속 작업 브리프 생성

검증 기준:
- 유료 모델 상태와 무관하게 시연 가능하다.
- 공개 링크 읽기가 실패하면 실패 사유를 보여주고, 사용자가 원문/메모를 넣어 보완할 수 있다.
- 실패하더라도 입력값 손실 없이 다시 시도할 수 있다.
- 복사된 브리프를 다른 문서나 메신저에 바로 붙여넣을 수 있다.

## Phase 5. 다음 확장 후보

목적:
라이브 이후 송이를 실제 업무 자동화 직원으로 키울 방향을 정리한다.

후보:
- 실제 웹 페이지 본문 수집
- 유료 AI 요약 옵션
- 이미지/썸네일 저장
- 경쟁사/레퍼런스 보드
- 후속 직원 배정 기능
- 대량 URL 업로드
- 리서치 결과를 Notion, Google Sheet, 콘텐츠 캘린더로 전송

주의:
- 유료 AI 분석은 비용 승인, 모델명, 예상 비용, 재시도 정책을 먼저 확정한 뒤 붙인다.
- 윤미 교체 직원명이 확정되기 전까지 송이 산출물은 범용 브리프로 유지한다.
