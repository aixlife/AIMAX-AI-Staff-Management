# Yeri/Hyunju Everyone Can Use Goal - 2026-05-30

## Decision

AIMAX 안정화의 최우선 목표는 "구매자 누구나 예리/현주를 쓸 수 있게 만드는 것"이다.

송이와 윤미는 웹기반 직원이라 상대적으로 잘 작동한다. 현재 가장 중요한 초점은 로컬 실행기, OS별 설치, 브라우저 자동화, 네이버 연결에 영향을 받는 예리/현주다.

이 목표는 단순히 내부 테스트 계정에서 한 번 성공하는 것이 아니다. 처음 사용하는 일반 구매자가 Mac 또는 Windows에서, 그리고 사용자의 PC 환경이 조금씩 달라도, 실제 배포 링크에서 최신 설치파일을 다시 다운로드한 뒤 스스로 설치하고 로그인하고 필요한 설정/연결을 마친 다음 예리/현주 작업을 생성해 네이버 입력과 임시저장까지 완료할 수 있어야 한다.

## Why This Matters

현재 일부 구매자는 아직 예리/현주를 한 번도 제대로 사용하지 못했다. 이 상태에서 기능 추가나 고도화보다 중요한 것은 신뢰 회복과 첫 사용 성공이다.

2026년 6월 1일까지 오류 수정 후 다시 다운로드해서 시도해 달라는 공지 팝업과 메일 안내는 이 목표를 위한 고객 커뮤니케이션이다. 문구는 변명보다 인정, 일정, 다음 행동, 추가 결제 없음, 책임지고 조치한다는 메시지를 우선한다.

## Guidance For Other AI Collaborators

Claude, Gemini, Codex, Windows AI 개발자는 예리/현주 안정화 작업에서 아래 기준을 우선한다.

- 내부 demo 성공보다 실제 구매자 첫 사용 성공을 우선한다.
- 송이/윤미 웹기반 기능보다 예리/현주 로컬 실행기 기반 기능의 cross-platform 안정화를 우선한다.
- Mac/server 작업과 Windows/local-agent 작업을 분리해서 기록하고 검증한다.
- "배포 완료"는 운영 서버 반영이나 소스 smoke 통과만으로 판단하지 않는다.
- 배포 후 실제 공개/운영 링크에서 다운로드한 설치파일로 설치부터 임시저장까지 통과해야 사용자 기준 테스트로 인정한다.
- 구매자에게 필요한 안내는 짧고 명확해야 한다: 지금 왜 안 되는지, 언제 다시 시도하면 되는지, 어디서 다시 다운로드하는지, 추가 결제 없이 사용 가능한지.
- 오류 보고는 사용자에게 책임을 넘기는 장치가 아니라, 예리/현주를 누구나 쓸 수 있게 만들기 위한 복구 루프다.
- 새 기능, UI 개선, 비용 최적화보다 설치, 연결, 설정, 실행, 임시저장 성공률이 우선이다.

## Verification Standard

예리/현주 안정화 완료 판단은 최소한 아래 증거를 요구한다.

- 운영 웹 UI에서 구매자와 같은 흐름으로 로그인한다.
- 실제 배포 링크에서 최신 macOS/Windows 설치파일을 다시 다운로드한다.
- 다운로드한 설치파일로 Mac과 Windows 각각에서 실제 설치 또는 설치본 기준 실행을 확인한다.
- 로컬 실행기 연결 상태가 웹에서 올바르게 보인다.
- AI/API 연결 또는 필요한 키 설정이 사용자가 이해할 수 있게 안내된다.
- 예리/현주 작업 생성 버튼이 권한/연결/설정 조건에 따라 명확히 활성화 또는 차단된다.
- 최소 유료 테스트가 필요한 경우, 사전 승인된 범위 안에서 1회만 실행하고 중복 재시도하지 않는다.
- 네이버 입력과 임시저장까지 완료되거나, 실패 시 사용자가 이해할 수 있는 오류와 관리자 진단 정보가 남는다.
- API/smoke/source-mode 테스트만으로는 완료 처리하지 않는다. 배포 후 실제 링크에서 받은 설치본과 실제 웹 UI 기준으로 확인해야 한다.

## Suggested Customer Notice Direction

공지 팝업과 메일의 핵심 문장은 다음 방향을 유지한다.

> 현재 일부 사용자 환경에서 정상 사용이 어려운 오류가 확인되어 2026년 6월 1일까지 수정 및 안정화 작업을 진행 중입니다. 6월 1일 이후 최신 버전을 다시 다운로드해 이용을 시도해 주세요. 기존 구매자는 추가 결제 없이 그대로 사용하실 수 있으며, 정상 사용 가능 상태가 되도록 끝까지 확인하겠습니다.
