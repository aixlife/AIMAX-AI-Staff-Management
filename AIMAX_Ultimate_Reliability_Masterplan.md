# [최종결정판] AIMAX 크로스 플랫폼 무중단 호환 및 극단적 미니멀 UI/UX 마스터플랜

> [!IMPORTANT]
> 본 설계도는 macOS와 Windows 양쪽 환경에서 **"100% 한 몸처럼 에러 없이 작동하는 크로스 플랫폼 호환성"**과, 복잡한 군더더기를 싹 걷어내고 **"딱 필요한 버튼만 있는 극단적 미니멀 UI/UX"**를 핵심 가치로 둡니다. 
> 시스템 기초 공사부터 AI 직원들의 세부 작동 메커니즘까지, 처음부터 다시 설계한다는 관점으로 꼼꼼하게 정리한 최종 대통합 설계도입니다.

## 1. 아키텍처 코어: 크로스 플랫폼 추상화 계층 (OS Abstraction Bridge)

Mac과 Windows는 파일 경로, 암호화 키체인, 하드웨어 식별자, 웹 드라이버 구동 방식이 완전히 다릅니다. 이 차이를 코드 레벨에서 직접 분기 처리하면 반드시 한쪽이 깨집니다. 
따라서 AIMAX 로컬 에이전트의 중심에 **OS 독립적 코어 브릿지(OS Abstraction Bridge)**를 세워 에러율을 제로(0)로 만듭니다.

```
                  ┌─────────────────────────────────────────┐
                  │          OS Abstraction Bridge          │
                  └────────────────────┬────────────────────┘
                                       │
         ┌─────────────────────────────┴─────────────────────────────┐
         ▼                                                           ▼
┌─────────────────────────────────┐                         ┌─────────────────────────────────┐
│           macOS Core            │                         │          Windows Core           │
├─────────────────────────────────┤                         ├─────────────────────────────────┤
│ • Security.framework Keychain   │                         │ • Windows Credential Manager    │
│ • IOPlatformUUID                │                         │ • Cryptography Registry UUID    │
│ • ~/.aimax-storage (POSIX)      │                         │ • %APPDATA%/aimax (UNC Path)    │
│ • Playwright Webkit/Chrome      │                         │ • Playwright Chromium/Edge      │
└─────────────────────────────────┘                         └─────────────────────────────────┘
```

### ① 로컬 자격증명 저장소의 통일 (`OSCredentialsBridge`)
- **문제**: macOS에서는 `keyring` 라이브러리가 키체인을 잘 호출하지만, Windows 11 권한 보안 정책 및 headless 상태에서는 에러를 뿜으며 평문 폴백 파일로 떨어짐.
- **해결**: 단일 통합 클래스 `OSCredentialsBridge`를 개발하여 OS를 감지하고 최적의 보안 저장소를 맵핑합니다.
  - **macOS**: `Security.framework`와 연동되는 시스템 Keychain 접근.
  - **Windows**: Win32 API 기반의 **Windows Credential Manager (DPAPI)** 및 `CryptProtectData` 인터페이스 호출.
  - **공통 폴백**: 키체인 접근 실패 시, 임의의 평문 텍스트 저장을 영구 금지하고 사용자 PC 고유 키를 기반으로 **PBKDF2 + AES-256-GCM** 암호화 후 SQLite 로컬 설정 테이블에만 저장.

### ② 고유 하드웨어 식별자 규격화 (`OSHardwareCompliance`)
- **문제**: IP, Wi-Fi Mac 주소 등은 네트워크 변경 시 계속 바뀌어 라이선스가 만료되는 참사 발생.
- **해결**: 절대로 하드웨어 자가 변경이 불가능한 OS 표준 UUID만 사용하도록 통일.
  - **macOS**: `ioreg -rd1 -c IOPlatformExpertDevice` 파싱 후 `IOPlatformUUID` 고정 추출.
  - **Windows**: 레지스트리 `HKEY_LOCAL_MACHINE\SOFTWARE\Microsoft\Cryptography\MachineGuid` 추출.

### ③ 크로스 플랫폼 자동 브라우저 드라이버 프로비저닝 (`OSAutoDriverResolver`)
- **문제**: 크롬/엣지 브라우저 버전이 업데이트될 때마다 로컬 셀레늄/플레이라이트의 드라이버 버전이 맞지 않아 실행 에러가 끊임없이 발생함.
- **해결**: 실행 시 드라이버를 직접 찾거나 수동 설치할 필요 없이, `OSAutoDriverResolver` 모듈이 OS 종류와 현재 브라우저 버전을 자동 감지(Runtime Dynamic Checking)하고, 알맞은 최신 바이너리 드라이버를 로컬 저장소 캐시(`/assets/drivers/`)에 무인 다운로드 및 자동 매칭하여 실행 안정성을 200% 확보함.

## 2. UI/UX 철학: 필수 기능만 남기는 미니멀 CTA(Call to Action) 프레임워크

복잡하고 현란한 버튼들과 지나친 로그 덤프는 사용자에게 혼란과 시스템 오작동 불안감만 안겨줍니다. 
AIMAX UI/UX는 **"진짜 직원을 부리듯 핵심만 지휘하는 3대 극단적 미니멀 UI"**로 리뉴얼합니다.

```
┌───────────────────────────────────────────────────────────────────────────┐
│  AIMAX Dashboard (Minimal Interface)                                      │
├───────────────────────────────────────────────────────────────────────────┤
│                                                                           │
│  [송이: 리서처] ➔ 🟩 [새 프로젝트 분석 시작] ➔ URL 붙여넣기 ➔ [분석완료 브리프 복사] │
│                                                                           │
│  [예리: 글쓰기] ➔ 🟧 [초안 검수 및 발행 승인] ➔ 확인 ➔ [최종 네이버 포스팅]       │
│                                                                           │
│  [현주: 이웃소통] ➔ 🟦 [이웃 맺기 가동] ➔ (에러/캡차 시) ➔ [보안숫자 수동 해결]  │
│                                                                           │
└───────────────────────────────────────────────────────────────────────────┘
```

1. **상태 중심의 동적 UI**: 항상 화면이 꽉 찬 복잡한 대시보드를 보여주지 않고, 에이전트가 "대기 중", "작업 중", "사용자 개입 필요" 상태일 때 꼭 눌러야 하는 **단 하나의 메인 버튼(CTA)**만 큼직하게 띄웁니다.
2. **원클릭 복사 & 내보내기**: 모든 복잡한 편집 화면을 스크롤하지 않고, 상단에 **[브리프용 원클릭 복사]**, **[텍스트 다운로드]** 버튼만 두어 즉시 재활용할 수 있게 가치를 완성합니다.
3. **직관적 오류/복구 배너**: 에러 발생 시 알 수 없는 프로그래밍 예외 코드를 뿜어내는 대신, 상단 배너에 **"네이버 비밀번호 변경 필요" [지금 설정하기]** 같은 형태로 해결을 위한 다이렉트 버튼만 노출합니다.

## 3. 7인 AI 스태프 미니멀 UI & 크로스 플랫폼 상세 설계

### ① 송이 (콘텐츠 리서치) - **Web-First (크로스 플랫폼 공통)**
*   **비즈니스 흐름**: 유저가 유튜브/인스타/틱톡 링크를 던지면, 숏폼을 분석해 기획 브리프로 만들어 로컬에 저장합니다.
*   **미니멀 UI 스크린 플로우 (딱 4개 버튼)**:
    1. **[새 프로젝트 만들기]** 클릭 (프로젝트명과 리서치 목표만 간단히 입력)
    2. **[영상 링크 등록]** 클릭 (멀티 라인 입력창에 링크를 쭉 복사-붙여넣기)
    3. **[분석 시작]** 클릭 (화면에는 `1단계: 자막 추출 중...` ➡️ `2단계: AI 구조 분석 중...` ➡️ `3단계: 로컬 저장 완료`와 같은 간결한 3단계 상태 바만 표시)
    4. 분석 완료 카드를 클릭하면 상세 모달이 뜨며, 우측 상단에 딱 하나의 **[제작 브리프 복사]** 버튼만 제공.
*   **크로스 플랫폼 호환**:
    - **Mac/Windows 공통**: 외부 스크래핑 IP 차단을 피하기 위한 Web-First 프록시 로테이션 서버 가동.
    - 다운로드한 영상 캐시, 썸네일, 전사 원문은 Mac의 경우 `~/Library/Application Support/Aimax/`, Windows의 경우 `%APPDATA%\Aimax\` 표준 분기 경로로 일괄 통합 매핑 저장.

---

### ② 예리 (블로그 본문 자동 포스팅) - **Hybrid (Mac/Windows)**
*   **비즈니스 흐름**: 작성된 양질의 글 초안을 검수한 뒤, 로컬 PC의 브라우저를 띄워 네이버 블로그에 포스팅을 집행합니다.
*   **미니멀 UI 스크린 플로우 (딱 2개 버튼)**:
    1. **[초안 확인 및 발행 승인]**: 예리가 글 작성을 마치면 임시 어드민 웹 화면에 초안이 로드되고 이 버튼이 활성화됨.
    2. **[최종 블로그 포스팅 실행]**: 사용자가 초안 승인을 누르면, 로컬 PC에서 예리가 기동하여 글쓰기를 시작함. (진행 상태는 `포스팅 중 [70%]...` 단일 진행률 바로 단순화)
*   **크로스 플랫폼 호환**:
    - macOS와 Windows 양쪽에서 Playwright가 headless가 아닌 **실제 브라우저(headed)를 안정적으로 로컬 구동**할 수 있게 `playwright install chromium` 무인 버전을 코어 브릿지에서 실행 및 예외 대응함.
    - 스마트에디터 레이아웃 변화에 따른 **Vision API 좌표 획득 폴백** 기능은 Mac/Windows 화면 픽셀 스케일링(Retina vs Standard DPI) 차이를 감지하여 좌표 보정을 정밀 자동 수행.

---

### ③ 현주 (서이추 및 소통) - **Local-Only (Mac/Windows)**
*   **비즈니스 흐름**: 타깃 블로거들을 찾아가 이웃을 맺고 공감 및 소통 댓글을 남겨 잠재고객을 육성합니다.
*   **미니멀 UI 스크린 플로우 (딱 2개 버튼)**:
    1. **[서로이웃 맺기 가동]**: 타겟 키워드 설정 후 이 버튼을 누르면 백그라운드로 작동 시작.
    2. **[보안문자(CAPTCHA) 입력 및 해결 완료]**: 네이버 봇 방지가 작동하여 캡차가 뜨면 현주는 일시 정지하고 화면에 캡차 이미지와 텍스트 박스를 띄움. 사용자가 문자를 치고 이 버튼을 누르면 작업 재개.
*   **크로스 플랫폼 호환**:
    - Windows의 경우 백그라운드 작업이 절전 모드에 의해 끊기는 현상이 잦으므로 `win32gui` / `SetThreadExecutionState` API를 호출해 **작업 실행 중 Windows 시스템의 강제 절전(Sleep mode)을 방지**하도록 예외 조치.
    - Mac의 경우 Caffeinate 프로세스를 백그라운드에서 임시 구동하여 무중단 처리 보장.

---

### ④ 윤미 (숏폼 대본 작성) - **Web-First (크로스 플랫폼 공통)**
*   **비즈니스 흐름**: 분석된 숏폼 트렌드나 기획 목적에 의거해 즉각 촬영 가능한 고품질 대본을 뱉어냅니다.
*   **미니멀 UI 스크린 플로우 (딱 2개 버튼)**:
    1. **[3가지 타입 대본 생성]** 클릭. (자동으로 후킹형, 정보형, 스토리텔링형 대본이 카드 형식으로 슬라이드 렌더링됨)
    2. 마음에 드는 카드를 고르고 **[대본 복사 및 촬영 가이드 복사]** 버튼 하나로 마무리.
*   **크로스 플랫폼 호환**: Pure Web-First 아키텍처이므로 브라우저가 있는 어떠한 환경(macOS, Windows, 심지어 모바일 Safari/Chrome)에서도 동일한 대본 퀄리티와 100% 동일한 화면 제공.

---

### ⑤ 나경 (기획/퍼널 분석) / ⑥ 현성 (디자인) / ⑦ 상수 (퍼널 분석) - **Web-First (Planned)**
*   **나경 (기획)**: **[1억 퍼널 최적화 전략서 생성]** 단 하나의 메인 버튼으로 전체 유입 시나리오 도출.
*   **현성 (디자인)**: **[카드뉴스 생성]** ➡️ 4장 그리드 중 1장 선택 ➡️ **[선택 에셋 다운로드]**로 정밀 압축.
*   **상수 (퍼널분석)**: **[매출 데이터 업로드(CSV/Excel)]** ➡️ 알아서 누락된 부분 보정 후 **[이상치 경고 확인]** ➡️ **[한 장 요약 보고서 복사]** 버튼 제공.
*   **크로스 플랫폼 호환**: 
    - 세 직원 모두 `staff-catalog.json` 선언적 카탈로그 구조를 공유하므로, 새로운 에이전트가 로드되거나 배포될 때 클라이언트 앱 버전 업데이트 없이도 즉시 Mac/Windows 전체 플랫폼에 UI 및 API 폼 스키마가 무인 동기화됨.

## 4. 교차 검증 (Cross-Verification) 프로토콜

작업이 완료된 후 Mac 개발 환경과 Windows 윈도우 에이전트 간의 **완벽한 교차 검증**을 위한 4단계 표준 체크리스트입니다.

```
[Mac 환경 (Server & Core-Bridge 기획)] 
                 ↕️ (Syncthing 대칭 검증)
[Windows 환경 (app.py & Windows UI 안정화)]
```

| 검증 단계 | 검증 항목 | Mac OS 검증 방법 | Windows 검증 방법 | 성공 판정 기준 |
| :--- | :--- | :--- | :--- | :--- |
| **1단계** | **Hardware ID 추출 독립성** | `aimax_compliance.py` 실행 시 `IOPlatformUUID` 값 획득 확인 | 레지스트리 `MachineGuid` 획득 확인 | Wi-Fi 끄기, VPN 연결 등 네트워크 변경 시에도 양쪽 식별자가 고정되는가 |
| **2단계** | **로컬 크레덴셜 암호화 저장** | `.env` 없이 키체인 정상 호출 확인 | DPAPI (`CryptProtectData`) 함수 정상 리턴 확인 | 암호화된 계정 정보가 평문 노출 없이 SQLite에 안전히 저장되는가 |
| **3단계** | **Playwright 브라우저 무인 구동** | `OSAutoDriverResolver`가 헤드리스/헤디드 크롬 정상 구동 | 동일 모듈이 윈도우용 크롬 드라이버 무인 다운로드 및 구동 | 버전에 맞는 웹 브라우저가 사용자 개입 없이 자동 실행되는가 |
| **4단계** | **미니멀 UI & 오류 알림 연동** | 스마트에디터 오류 강제 유도 시 에러 배너 및 [재시도] 버튼 정상 노출 | 윈도우 로컬 GUI에 캡차 수동 입력 대기 창 정상 활성화 | 크래시 없이 실행이 일시 정지(HITL)되고 원클릭 복구가 가능한가 |

---

## 5. 결론 및 비즈니스 전환

이 대통합 설계는 **"사용자가 다른 복잡한 기능에 머리 아프지 않고 오직 제작 가치(브리프 복사, 포스팅 승인)에만 집중하도록 돕는 극도의 미니멀리즘"**을 지향합니다. 동시에 복잡하고 깨지기 쉬운 로컬 환경의 장벽들을 **OS Abstraction Bridge**를 통해 macOS와 Windows 양측 모두에서 극복합니다.
