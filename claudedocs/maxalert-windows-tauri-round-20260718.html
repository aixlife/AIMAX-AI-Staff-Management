선결 확인 3건 — 전부 실측으로 답

1. Tauri NSIS의 Electron 대체 — 된다

  • 배포된 v0.1.17의 electron-updater는 latest.yml sha512 검증 후 새 설치기를 --updated /S --force-run으로 실행 (소스 실독). 배포본 app-update.yml에 publisherName 없음 → 무서명 Tauri NSIS 수용 (AIXLIFE 실물 확인).
  • 신규 installer-hooks.nsh: PREINSTALL은 앱 종료만, Electron 제거·잔재 정리·바로가기 복원·재실행은 새 파일 설치 성공 후(POSTINSTALL) — 설치 중간 실패 시 구버전 생존.

2. 데이터 마이그레이션 — 기존 구현 + 원자화 보강

  • Store::load가 첫 실행에 %APPDATA%\maxalert → com.makefriends.maxalert 복사 (v0.2.1 때 구현). 이번에 파싱 검증 + tmp→rename 원자화, 저장도 원자화.

3. 빌드 경로 — GH Actions 채택

  • 리포 PUBLIC(무료)+WRITE, AIXLIFE엔 Rust 없음. push마다 NSIS+latest.yml+SHA256 아티팩트, 릴리스 게시는 수동(사람 결정). 파트너도 Rust 설치 없이 기존 릴리스 흐름 유지.

Codex Sol 반박 리뷰 13건 판정

  • 실측 기각 1: 서명 검증 차단설 — publisherName 부재 실확인.
  • 수용 9: 설치 순서 재설계(제거를 설치 성공 뒤로)·spawn 후 자진 종료 제거·사이렌 발화 중 연기·저장/마이그레이션 원자화·native roots+시스템 프록시·RunAsUser 재실행 등 전부 반영.
  • 부분 수용 2: URL https/루프백 제한, 랜덤 임시파일+create_new.
