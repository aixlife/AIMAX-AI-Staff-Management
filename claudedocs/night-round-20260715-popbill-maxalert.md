## 2. 맥스(MaxAlert) 알림 폭주 — 원인 확정

### 증상
아침에 실행하자마자 사이렌 연쇄 발화, 정지 불가, 강제 종료도 어려움.

### 원인 (실데이터 + 설치 앱 소스로 재구성)
1. **노션 개인 브리지**(launchd `com.aixlife.maxalert.personal-sync`, 매일 08:00)가 노션 일정 DB+업무 DB에서 미완료 태스크 최대 12건을 매일 가져와 **오늘 날짜 + 30분 간격 인공 dueAt 슬롯**(09:00~14:30대)으로 주입
2. 노션에서 완료 처리 안 된 태스크는 **다음날 또 새 항목으로 재주입** — "이전 게 안 지워지고 계속 남는다"는 민수님 추정이 브리지 레벨에서 정확히 맞음 (7/13엔 24건, 7/14엔 13건, 누적 75건 중 미완료 59건)
3. 앱 사이렌 설계: 기한 3분 전부터 `완료 or 미루기 외엔 끌 수 없는` 전면 kiosk 사이렌 + 700ms마다 전 모니터 포커스 강탈 + 하나 닫히면 다음 대상 즉시 발화(연쇄)
4. 시뮬레이션(7/14 실데이터): 09:30 실행 → 2건 연쇄, 11:00 → 5건, 14:00 → **11건 연쇄 대기**. 여기에 30분마다 새 슬롯 도래
5. 데이터 파일 mtime이 브리지 기록(08:04) 그대로 = 그날 완료/미루기 클릭이 하나도 저장 안 됨 — 포커스 강탈(700ms 주기)이 클릭을 씹었을 가능성 + 강제 종료

과거 날짜 누적분(46건)은 date==today 필터 때문에 직접 사이렌을 울리진 않지만, 매일 재주입이 문제의 본체.

### 수정 제안 (3단)
- **A. 즉시(데이터)**: 과거 미완료 46건 ackDue 일괄 마킹 스크립트 준비 완료(백업+멱등) — 승인 시 실행
- **B. 브리지(sync.mjs)**: [태스크]는 dueAt 슬롯 없이 포스트잇 표시만(사이렌 제외), 사이렌은 실제 시간 있는 [일정]만. 과거 날짜 미완료 브리지 항목 자동 정리 추가
- **C. 앱(리빌드 필요, 별도 라운드)**: "오늘 그만 울리기" 전체 정지 버튼, 연쇄 사이렌 상한, 포커스 강탈 주기 완화(클릭 씹힘 수정)

A+B만으로 "실행하면 난리" 상태는 해소됨. 승인 대기.
