# 예리(블로그 직원) 네이버 블로그 리서치 통합 스펙

작성일: 2026-07-10
상태: 대기 (다음 AIMAX 세션에서 착수)

## 목표

앱 사용자가 자기 네이버 블로그 주소를 등록하면, 예리가 그 블로그의 기존 글 원문을 수집해 사용자 문체 프로필을 만들고, 이후 글 작성 시 그 문체를 반영한다. 민수 개인용이 아니라 앱 배포 사용자 전원이 쓰는 제품 기능이다.

## 배경

- k-skill의 `naver-blog-research` 스킬(MIT)이 네이버 블로그 검색·원문 읽기·이미지 다운로드를 비로그인 공개 표면으로 수행한다. 참조 사본: 이 프로젝트 `.claude/skills/naver-blog-research/` (SKILL.md + scripts).
- 이 앱은 이미 `codex-skills/songi-local` 패턴으로 직원(송이)에게 스킬을 배포한 전례가 있다. 예리도 같은 경로가 유력하다.
- 발행 전 문체 검수용으로 `korean-humanizer` 스킬도 `.claude/skills/`에 참조 사본이 있다.

## 통합 방식 제안 (songi-local 패턴)

0. 착수 시 `docs/runbooks/aimax-employee-release.md`(Employee Launch Checklist)를 먼저 읽고 web-first / local-agent-required / hybrid를 결정한다. 블로그 수집은 서버 IP 차단 가능성과 사용자 로컬 데이터 원칙 때문에 local-agent 쪽이 유력하지만, 체크리스트 기준으로 판단한다.
1. `naver-blog-research` 스크립트를 `codex-skills/naver-blog-research/`로 이식하되, 앱 빌드에 codex-skills가 자동 포함되는지 먼저 확인한다. 포함된다면 검증 완료 전에는 넣지 않는다.
2. 예리 직원 정의에 "블로그 문체 학습" 도구/단계를 추가: 사용자 블로그 주소 입력 → 글 목록 수집 → 원문 수집 → 문체 프로필 문서 생성(어휘·문장 길이·어미·구성 패턴) → 예리 시스템 프롬프트에 주입.
3. 글 작성 플로우 마지막에 humanizer식 검수 단계(AI 티 제거)를 선택 옵션으로 추가.

## 제약 (필수)

- 비로그인 공개 수집만. 로그인 세션·차단 우회 금지.
- 기본값은 사용자 본인 블로그. 타인 블로그 문체 모사는 기능으로 제공하지 않거나 명시 고지 후 제한.
- 요청 속도 제한 준수, 429/차단 시 재시도 대신 중단·안내.
- 수집 원문은 사용자 로컬에만 저장. 앱 서버로 업로드하지 않는다.
- Windows/Mac 양쪽에서 스크립트 실행 검증 필수 (배포 대상이 양 플랫폼).
- 업스트림 출처(k-skill, MIT) 표기 유지.

## 검증 기준

1. 실제 네이버 블로그 1개(글 10개 이상)로 수집 → 문체 프로필 생성이 Windows/Mac 모두에서 성공.
2. 문체 프로필 반영 전/후 예리 산출물 비교에서 사용자가 차이를 인지.
3. 블로그 주소 오입력·비공개 블로그·글 0개 케이스에서 안전하게 실패.
4. 빌드 산출물에 불필요한 스크립트/의존성이 새로 들어가지 않았는지 확인.
