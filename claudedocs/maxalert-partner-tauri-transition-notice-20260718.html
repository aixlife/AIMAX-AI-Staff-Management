[맥스 윈도우 Tauri 전환 안내] 안녕하세요, 민수입니다. 맥스 관련 큰 변경 하나 공유드리고, 협의드릴 것이 있어 연락드립니다. 1. 결정 내용 맥 버전이 이미 Tauri 기반(v0.2.x)으로 운영 중인데, 윈도우도 같은 Tauri 소스로 통합하기로 했습니다. 다음 릴리스가 통합 버전 v0.2.2입니다. 설치 파일이 96MB에서 16.5MB로 줄고, 맥/윈도우가 한 코드베이스가 됩니다. 2. 기존 사용자 전환 지금 쓰시는 릴리스 채널(maxalert-releases) 그대로 v0.2.2를 올리면, 기존 Electron 설치본들이 자동업데이트로 Tauri 버전으로 교체됩니다. 포인트·스트릭·설정 데이터는 자동 마이그레이션되고, Electron 설치본·자동시작 잔재는 설치기가 정리합니다. 이 전환 체인은 실기 검증을 전부 마친 뒤에만 릴리스합니다. 문제가 생겨도 같은 채널로 픽스 버전을 올리면 사일런트 자동업데이트로 즉시 배포됩니다 (지금과 동일). 3. 빌드 방식 변경 electron-builder 로컬 빌드 대신 GitHub Actions가 push마다 NSIS 설치기와 latest.yml을 자동 생성합니다. Rust를 설치하실 필요 없이 지금처럼 renderer(HTML/JS) 작업을 하시면 되고, 릴리스는 Actions 산출물을 maxalert-releases에 올리는 기존 방식 그대로입니다. 4. 부탁드릴 것 (전환 기간 피처 프리즈) 전환 릴리스가 나갈 때까지 master에 새 기능 커밋을 잠시 보류 부탁드립니다. 전환 중 master에 올라온 기능은 Tauri 쪽에 다시 이식해야 해서 이중 작업이 됩니다. 급한 수정이 있으면 먼저 말씀 주세요 — 전환 브랜치에 바로 반영하겠습니다. 예상 기간은 실기 검증 포함 며칠 내입니다. 5. 이후 개발 방식 renderer 쪽(화면·위젯·사이렌 UI)은 기존 파일 구조가 거의 그대로 유지됩니다. main.js에 있던 네이티브 로직(창 관리·트레이·스케줄러·노션 동기화)은 Rust로 이식돼 있습니다. 기능 방향 결정은 지금처럼 윈도우 쪽 기준을 우선하니, 네이티브 쪽 손이 필요한 기능은 같이 상의해서 진행하면 좋겠습니다. 궁금한 점이나 우려되는 부분 있으면 편하게 말씀 주세요.
