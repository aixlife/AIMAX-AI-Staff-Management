1. 시작 전 확인사항
--------------------------------------------------

  아래 항목만 미리 준비되어 있으면 됩니다.

  ① 구글 크롬 브라우저 (필수)
     없으면 실행 시 자동으로 다운로드 페이지가 열립니다.
     다운로드: https://www.google.com/chrome/

  ② Python은 없어도 됩니다 — 자동으로 설치됩니다.
     (Mac: Homebrew 경유 / Windows: winget 경유)


--------------------------------------------------
2. 프로그램 실행 방법
--------------------------------------------------

  ┌─────────────────────────────────────────────┐
  │  Mac   →  NaverBlogAuto.command  더블클릭   │
  │  Windows →  setup.bat  더블클릭             │
  └─────────────────────────────────────────────┘

  [ 처음 실행할 때 ]
    자동으로 아래 순서가 진행됩니다.
      1) 크롬 설치 확인
      2) Python 설치 확인 (없으면 자동 설치, 수 분 소요)
      3) 필요한 패키지 설치 (2~5분 소요)
      4) 프로그램 실행

    ※ Mac 최초 실행 시 접근성 권한 설정이 필요합니다.
       터미널에 안내 메시지가 표시됩니다.
       시스템 설정 → 개인정보 보호 및 보안 → 접근성
       → 터미널을 목록에 추가하고 허용

    ※ Windows에서 Python 자동 설치 시
       관리자 권한 팝업이 뜨면 [예]를 눌러주세요.
       설치 완료 후 setup.bat을 한 번 더 실행해주세요.

  [ 두 번째 실행부터 ]
    설치가 이미 완료되어 있으면
    NaverBlogAuto.command (Mac) 또는 setup.bat (Windows)을
    더블클릭하면 바로 프로그램이 실행됩니다.


--------------------------------------------------
3. 첫 설정 (계정 & API 키)
--------------------------------------------------

  프로그램 첫 실행 시 안내 팝업이 자동으로 뜹니다.
  Settings 탭에서 아래 정보를 입력하고 [설정 저장]을 클릭하세요.

  ① 네이버 ID     : 네이버 로그인 아이디
  ② 비밀번호      : 네이버 로그인 비밀번호
  ③ Gemini API Key: AI 글 생성에 사용 (Gemini 선택 시)
  ④ Claude API Key: AI 글 생성에 사용 (Claude 선택 시, 추천)

  ─ API 키 발급 방법 ─
  • Claude : https://console.anthropic.com → API Keys → Create Key
  • Gemini : https://aistudio.google.com   → Get API key

  ※ 입력한 비밀번호와 API 키는 OS 보안 저장소(키체인)에
    암호화되어 저장됩니다. 외부 유출 걱정 없습니다.


--------------------------------------------------
4. 기능 사용 방법
--------------------------------------------------

  [ Write — 블로그 글 자동 작성 ]
    1. Write 탭 클릭
    2. 키워드 입력 (예: 서울 맛집 추천)
    3. 글 스타일 선택: 정보성 / 구매성 / 광고성
    4. AI 모델 선택 (Claude 추천)
    5. [글쓰기 시작] 클릭
    → AI가 자동으로 글 작성 후 네이버 블로그에 발행

  [ Bulk Post — 대량 발행 ]
    엑셀 파일에 키워드 목록을 작성하여 한꺼번에 발행
    ※ 엑셀 양식: A열=키워드, B열=스타일, C열=카테고리

  [ Like / Comment / Neighbor ]
    이웃 블로그 공감, 댓글, 서로이웃 신청 자동화
    ※ 횟수를 너무 높게 설정하면 계정 제재 위험이 있습니다.

  [ Neighbor 탭 전용 옵션 ]
    • 신청 멘트    : 줄바꿈으로 여러 멘트 입력 → 매번 랜덤 선택
                     {닉네임}은 블로그 ID로 자동 치환됩니다.
    • 속도 모드    : 안전(30~90초/건) / 보통(4~15초) / 빠름(3~8초)
    • Cool-down    : N명마다 60~180초 휴식 (봇 탐지 회피)
    • 일일 상한    : 네이버 공식 100명 기준, 기본값 80명 권장


--------------------------------------------------
★★ 서로이웃 자동 신청 — 차단 위험 고지 (반드시 읽기) ★★
--------------------------------------------------

  이 기능은 계정 제재 위험이 가장 큰 기능입니다. 반드시 아래 안내를
  숙지한 후 사용하세요.

  ① 네이버 자동화 탐지 강화 (2025년 7월~)
     • 네이버는 서이추/공감/댓글 자동화 행위에 대한 탐지를 대폭 강화
       했습니다. Selenium 기반 도구도 탐지 대상입니다.
     • 탐지 시: 저품질 블로그 지정, 서이추 기능 30일 정지,
       심한 경우 블로그 영구 정지까지 발생합니다.

  ② 네이버 공식 한도
     • 하루 서로이웃 신청: 최대 100명
     • 공식 한도를 맞춰도 '패턴'이 부자연스러우면 차단됩니다.
     • 본 프로그램 기본값: 80명 (공식 대비 20% 버퍼)

  ③ 권장 사용 프로파일
     • 첫 1주일 : 안전 모드 + 일일 30명 이하로 시작
     • 2~3주차 : 계정 반응 관찰 후 점진 증량 (50명 → 80명)
     • 절대 금지: 빠름 모드 + 일일 80명 초과 (즉시 탐지 위험)

  ④ 멘트 작성 원칙
     • 동일 멘트 반복은 봇 탐지 핵심 신호입니다.
     • 최소 3개 이상의 멘트를 준비하세요.
     • {닉네임} 변수를 반드시 포함하여 개인화하세요.
     • 일반적인 "소통해요", "방문해주세요"는 낮은 반응률입니다.

  ⑤ 분산 실행 권장
     • 하루치를 한 번에 몰아서 신청하지 마세요.
     • 오전/오후로 나눠서 2회 실행 권장 (각 30~40명).
     • Cool-down 기능을 끄지 마세요 (기본 10명마다 휴식).

  ⑥ 계정 상태 관찰
     • 신청 후 3~7일 내 '서로이웃 수락률'이 10% 미만이면 중단
     • '블로그 유입' 지표가 급락하면 저품질 의심 — 자동화 중지
     • 검색 노출이 사라지면 이미 저품질 상태일 가능성

  ⑦ 법적/도의적 책임
     • 본 도구 사용은 네이버 이용약관에 저촉될 수 있습니다.
     • 계정 제재, 블로그 영구 손상, 저품질 지정 등의 모든 책임은
       사용자 본인에게 있습니다.
     • 상업적 어뷰징/홍보 스팸 목적 사용을 금지합니다.

  ⑧ 일일 카운터 자동 저장
     • 프로그램이 하루 신청 횟수를 자동 누적합니다.
     • 일일 상한 도달 시 작업이 중단됩니다.
     • 자정 이후 자동 리셋 (네이버 ID별 분리 저장).


--------------------------------------------------
5. ★ 주의사항 — 반드시 읽어주세요
--------------------------------------------------

  ① 계정 보안
     • 네이버 계정은 2단계 인증을 사용하지 마세요.
       (자동 로그인이 불가능합니다)
     • API 키는 타인과 절대 공유하지 마세요.
       (비용이 청구될 수 있습니다)

  ② 네이버 정책 준수
     • 하루 과도한 자동화 활동은 계정 제재(블로그 정지)로
       이어질 수 있습니다.
     • 권장 일일 포스팅: 계정당 3~5건 이하
     • 권장 공감/댓글: 하루 20~30건 이하
     • 신규 계정은 2~4주 워밍업(직접 활동) 후 자동화 시작

  ③ 콘텐츠 품질
     • AI가 생성한 글은 발행 전 한 번 검토하는 것을 권장합니다.
     • 동일 내용을 여러 계정에 그대로 복사하면 스팸으로
       감지될 수 있습니다.

  ④ 프록시 사용 권장 (멀티계정 운영 시)
     • 여러 계정을 같은 IP로 운영하면 계정 연결 감지 위험
     • 계정마다 다른 주거용 프록시 사용 권장

  ⑤ CAPTCHA 발생 시
     • 로그인 시 CAPTCHA가 뜨면 프로그램이 알려줍니다.
     • 브라우저에서 직접 로그인 후 다시 시도하세요.
     • 같은 IP에서 반복 시도를 피하고 잠시 대기 후 재시도.

6. 자주 묻는 질문 (FAQ)
--------------------------------------------------

  Q. 프로그램이 처음 실행될 때 너무 오래 걸려요.
  A. 처음 한 번만 패키지 설치(2~5분)가 진행됩니다.
     두 번째 실행부터는 바로 켜집니다.

  Q. Mac에서 더블클릭해도 아무 반응이 없어요.
  A. NaverBlogAuto.command 파일을 우클릭 → [열기]를 선택하세요.
     "개발자를 확인할 수 없음" 팝업이 뜨면 [열기]를 클릭.
     이후부터는 더블클릭으로 바로 실행됩니다.

  Q. Windows에서 Python 설치 후에도 안 된다고 해요.
  A. setup.bat을 한 번 더 실행해주세요.
     Python 설치 직후에는 PATH 적용을 위해 재실행이 필요합니다.

  Q. 로그인이 계속 실패해요.
  A. ① Settings 탭에서 아이디/비밀번호 재확인
     ② 네이버 직접 로그인 후 "낯선 기기" 허용 처리
     ③ 2단계 인증이 켜져 있으면 해제 필요

  Q. "크롬이 없습니다" 오류가 나요.
  A. https://www.google.com/chrome/ 에서 크롬 설치 후 재실행

  Q. API 키를 입력했는데 글 생성이 안 돼요.
  A. ① 키 앞뒤 공백 제거 확인
     ② Claude: 크레딧 잔액 확인 (console.anthropic.com)
     ③ Gemini: API 활성화 상태 확인 (aistudio.google.com)

  Q. Mac에서 이미지 업로드가 안 돼요.
  A. 시스템 설정 → 개인정보 보호 → 접근성에서
     터미널 허용 여부 확인

  Q. 프로그램을 업데이트하려면 어떻게 하나요?
  A. 새 버전 파일을 덮어씌우고 setup.bat 또는
     NaverBlogAuto.command를 다시 실행하면 됩니다.
